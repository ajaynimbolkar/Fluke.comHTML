Understanding Containers

Usually all content that is in a page lives inside a div with a class of .container. This container has a fixed width on desktop medium and desktop large (md & lg) and 100% width on mobile and desktop (xs & sm). The container is attached to the custom breakpoints that have been defined for Fluke.

When you want the background color of the container to extend to the edge of the screen, you have to wrap the .container div inside another div with a class that has a background color. This happens often enough that we have defined the class of .bg-gray

Fluid Containers

We can use .container-fluid instead of .container when we want to expand to fill the whole width of the screen. This is rarely used for Fluke content, but you will likely need them for the occassional banner.

Fluke Breakpoints

We have defined custom breakpoints for this build, just a little different than the default Bootstrap breakpoints.

Because bootstrap is “mobile first” when we style for the default range, it styles everything after as well. Essentially the styling flows from smallest to largest, unless we put in a media query. If we style for default, and then add a media query for “md”, the default style will apply to “xs”, and “sm”, while the “md” style will style apply to “md” and “lg”. Make sense? Understanding how “mobile first” works is the most important concept to understand. Everything else is copy and paste.

  • Mobile: 0px-767px (Called “extra small”, uses “xs” in class names, this also lives in a 100% width container.)
  • Tablet: 768px-1032px (Called “small”, uses “sm” in class names, this also lives in a 100% width container.)
  • Desktop-Medium: 1033px-1279px (Called “medium”, uses “md” in class names. This lives in a fixed width container of 972px + 30px to account for the grid gutter width)
  • Desktop-Large: 1270px-∞ (Called “Large” uses “lg” in class names. (Lives in a fixed width container of 1230px +30px to account for the gutter width)

The relevent media queries themselves:

Mobile

/* There is no media query for mobile, because the default styles apply there*/

Tablet

/* Small devices (tablets, 768px and up) */ @media (min-width: @768) { ... }

Desktop Medium

/* Medium devices (desktops, 1025px and up) */ @media (min-width: @1025) { ... }

Desktop Large

/* Large devices (large desktops, 1280px and up) */ @media (min-width: @1280) { ... }
